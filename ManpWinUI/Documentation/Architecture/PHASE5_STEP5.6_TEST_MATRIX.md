## Test Procedure (Per Fractal)

For each fractal tested:

1. **Launch app** (if not already running)
2. **Open Fractal Browser panel**
3. **Navigate to category** (e.g., "Burning Ship Variants")
4. **Select fractal** (e.g., "BurningShipCubic")
5. **Verify parameters load correctly:**
   - Check Parameters tab shows correct controls
   - Check default values match expected
6. **Test toolbar → parameters sync:**
   - Change `MaxIterations` in toolbar (e.g., 50 → 500)
   - Change `CenterX` / `CenterY` (pan the view)
   - Change `Zoom` (zoom in/out)
   - If Julia-enabled: Change `JuliaCX` / `JuliaCY`
7. **Click Render button**
8. **Verify:**
   - ✅ Fractal renders with NEW values (not defaults)
   - ✅ Toolbar changes affected the render
   - ✅ No exceptions/errors in Output window
9. **Test parameter persistence:**
   - Close app
   - Relaunch app
   - Select same fractal
   - Verify last values were restored

---

## Success Criteria

**PASS if:**
- ✅ All tested fractals render correctly with toolbar values
- ✅ Toolbar changes propagate to render (not using defaults)
- ✅ No crashes, exceptions, or sync errors
- ✅ Parameter persistence works (values survive app restart)

**FAIL if:**
- ❌ Fractal renders with defaults (ignoring toolbar changes)
- ❌ Toolbar changes don't affect render
- ❌ Exceptions in Output window
- ❌ Parameters don't persist
